title: "Users and roles"
description: "Manage user accounts and understand what each role can do in Salesianos FC."
---

The **Users** page (`/admin/users`) lets you create and manage user accounts, assign roles, reset passwords, and link users to player profiles. Only users with the `admin` role can access this page.

## Roles

Salesianos FC has four roles. Assign the most restrictive role that covers the user's responsibilities.

| Role | What they can do |
|---|---|
| `admin` | Full access to all features, including user management (`/admin/users`), branding (`/admin/branding`), and the audit log (`/admin/audit`). Can perform every action available to other roles. |
| `delegado` | Create and edit players, matches, tournaments, rivals, and series. View all read-only features across the app. Cannot access administration pages. |
| `tesorero` | Full access to the treasury module: payments, fee rules, and fee status generation. Can view players, matches, and tournaments. Cannot create or edit non-treasury data. |
| `jugador` | View their own player profile and upcoming matches. Register their own payments (self-service). Cannot view other players' data or any management pages. |

<Note>
A `jugador` user must be linked to a player profile (via `player_id`) before they can see their own profile page.
</Note>

## User fields

Each user account has the following fields:

| Field | Type | Description |
|---|---|---|
| `id` | string (UUID) | Unique identifier for the user. |
| `username` | string | Login name. Must be unique across all users. |
| `role` | `admin` \| `delegado` \| `tesorero` \| `jugador` | The user's role. Determines what they can access. |
| `active` | boolean | Whether the user can log in. Set to `false` to deactivate without deleting. |
| `created_at` | ISO 8601 datetime | When the account was created. |
| `updated_at` | ISO 8601 datetime | When the account was last modified. |
| `player_id` | string (UUID) \| null | Optional link to a player profile. Enables the `jugador` role to access their own data. |

## Creating a user

Only `admin` users can create new accounts.

<Steps>
  <Step title="Open the create dialog">
    On the **Users** page, click **Nuevo usuario**. The create dialog opens.
  </Step>
  <Step title="Enter access credentials">
    Fill in **Usuario** (username) and **Contraseña** (password). Both fields are required.
  </Step>
  <Step title="Select a role">
    Choose a role from the **Rol** dropdown: `admin`, `delegado`, `tesorero`, or `jugador`.
  </Step>
  <Step title="Optionally link to a player">
    Select a player from the **Vincular a jugador** dropdown to connect this account to a player profile. Leave it blank if the user is not a player. A player can only be linked to one account at a time.
  </Step>
  <Step title="Save">
    Click **Crear**. The new user appears in the list immediately.
  </Step>
</Steps>

<Tip>
When creating an account for a player, set the role to `jugador` and link their player profile in the same step. This gives them access to their own data without exposing any management features.
</Tip>

## Resetting a password

You can reset any user's password at any time without knowing their current password.

1. In the user list, click the **key icon** (Cambiar contraseña) next to the user.
2. Enter the new password in **Nueva contraseña**. Minimum 8 characters.
3. Repeat it in **Repetir contraseña**.
4. Click **Guardar**.

<Warning>
There is no confirmation email. The password changes immediately. Make sure to communicate the new password to the user through a secure channel.
</Warning>

## Activating and deactivating users

To prevent a user from logging in without deleting their account, set their `active` field to `false`. Send a `PATCH /api/auth/users/:id` request with `{ "active": false }`.

Reactivate by sending `{ "active": true }`.

Deactivated users are shown as **Inactivo** in the user list. Their data and links to player profiles are preserved.

## Linking a user to a player

The `player_id` field connects a user account to a player profile. This is what allows a `jugador` user to see their own profile, fee status, and match schedule.

To link or change the linked player after account creation:

1. Click the **link icon** (Vincular jugador) next to the user in the list.
2. Select a player from the **Jugador** dropdown, or choose **— Ninguno (desvincular) —** to remove the link.
3. Click **Guardar**.

<Note>
Each player profile can only be linked to one user account. If you try to link a player who is already linked to another account, the API will return an error.
</Note>

## API endpoints

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/api/auth/users` | List all user accounts. |
| `POST` | `/api/auth/users` | Create a new user. |
| `PATCH` | `/api/auth/users/:id` | Update a user (role, active status, `player_id`, etc.). |
| `POST` | `/api/auth/users/:id/set-password` | Reset a user's password. Body: `{ "new_password": "..." }`. |

### Create user request body

```json
{
  "username": "carlos",
  "password": "supersecret",
  "role": "jugador",
  "active": true,
  "player_id": "a1b2c3d4-..."
}
```

`player_id` is optional. All other fields are required.
