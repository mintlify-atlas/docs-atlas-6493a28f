<Tip>
  You need a running instance of [salesianos_backend](https://github.com/NoahSPA/salesianos_backend) before you can log in. Start the backend first, then follow the steps below.
</Tip>

## Prerequisites

- **Node.js** 20 or later (required by the `engines` field in `package.json`)
- **npm** (bundled with Node.js)
- A running **salesianos_backend** instance with at least one admin user

## Steps

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/NoahSPA/salesianos_frontend.git
    cd salesianos_frontend
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```
  </Step>

  <Step title="Configure environment variables">
    Copy the example environment file and set your backend URL.

    <CodeGroup>
      ```bash macOS / Linux
      cp .env.example .env
      ```
      ```bash Windows
      copy .env.example .env
      ```
    </CodeGroup>

    Open `.env` and set `VITE_API_BASE` to your backend URL:

    ```env .env
    VITE_API_BASE=http://localhost:8000
    ```

    <Note>
      If you omit `VITE_API_BASE`, the app defaults to `http://localhost:8000`. For any other backend address, this variable is required.
    </Note>
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run dev
    ```

    Vite starts the dev server at **http://localhost:5173** with hot module replacement enabled.
  </Step>

  <Step title="Log in">
    Open `http://localhost:5173` in your browser. You will see the login page.

    Enter the **username** and **password** for an admin account created in your backend. After a successful login, the app loads the dashboard.

    <Tip>
      If you see a yellow banner saying "Servicio temporalmente no disponible", the app cannot reach the backend. Check that the backend is running at the URL in `VITE_API_BASE`.
    </Tip>
  </Step>
</Steps>

## Available scripts

| Script | Command | Description |
|--------|---------|-------------|
| `dev` | `vite` | Start the development server with hot reload |
| `build` | `tsc -b && vite build` | Type-check and build for production |
| `start` | `serve -s dist -l 8080` | Serve the production build on port 8080 |
| `preview` | `vite preview` | Preview the production build locally |
| `lint` | `eslint .` | Run ESLint across the project |
