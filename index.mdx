## Getting started

<Steps>
  <Step title="Clone and install">
    Clone the repository and install dependencies.

    ```bash
    git clone https://github.com/NoahSPA/salesianos_frontend.git
    cd salesianos_frontend
    npm install
    ```
  </Step>
  <Step title="Configure environment">
    Copy the example environment file and point it at your backend.

    ```bash
    cp .env.example .env
    # Edit .env and set VITE_API_BASE to your backend URL
    ```
  </Step>
  <Step title="Start the dev server">
    Launch the Vite development server with hot reload.

    ```bash
    npm run dev
    ```

    The app is available at `http://localhost:5173`.
  </Step>
  <Step title="Log in">
    Open the app in your browser and sign in with your admin credentials. Your backend must be running and reachable at the URL set in `VITE_API_BASE`.
  </Step>
</Steps>

## Roles at a glance

Salesianos FC uses four built-in roles that determine what each user can see and do.

| Role | Capabilities |
|------|-------------|
| **admin** | Full access — players, matches, tournaments, treasury, users, branding, audit log |
| **delegado** | Create and edit players, matches, tournaments, rivals, and series |
| **tesorero** | Manage treasury: validate payments, define fee rules, export reports |
| **jugador** | View their own profile, see upcoming matches, submit self-service payment registration |

<Note>
  Every protected route requires an active session. Users are automatically redirected to the login page when their session expires.
</Note>
